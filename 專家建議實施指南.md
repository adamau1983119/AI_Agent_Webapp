## 🎯 關鍵實施要點

### 1. 資料來源冗餘機制

**分級強制策略**:
- ✅ 事實類（日期、地址、價格、名次）→ 強制雙來源一致
- ✅ 趨勢類 → 建議性檢查，標註「單來源趨勢」

**健康度監控**:
```python
# 每5分鐘探測
health_score = calculate_health_score(source)
if health_score < 0.6:
    switch_to_backup(source)
if health_score < 0.4:
    enable_degraded_mode(source)  # 減少生成次數 + 延長重試間隔
```

**Fallback 標記**:
```python
content["fallback"] = True
content["fallback_badge"] = "Fallback 生成"
content["exposure_weight"] *= 0.7  # 降權30%
```

---

### 2. NLP + CV 交叉檢查

**分層閾值**:
- 核心要素（品牌、品項、明確詞）: ≥ 0.85
- 非核心要素（風格、氛圍）: ≥ 0.75

**模型選型**:
- 基礎: CLIP/BLIP2
- 微調: 領域詞彙嵌入庫（前200-500高頻詞）

**人工 Fallback SLA**:
- 常規審核: ≤ 24小時
- 高優先: ≤ 2小時
- 自動生成審核包（文本片段、疑點標記、候選照片Top-N）

---

### 3. 冷啟動策略

**問卷設計**:
- 可選（不強制）
- 3-5題，30秒完成
- 完成可解鎖「更準配方」

**靜默探測**:
- 連續3次不互動 → 提高探索比例 +20%，彈窗詢問
- 連續6次不互動 → 頻率降檔（3→1），標記「低響應用戶」

**權重設計**:
- 使用指數衰減: α = 0.7 → 0.3
- 公式: `weight = α * e^(-decay_rate * days)`

---

### 4. 品質控制

**雙閾值門檻**:
- ≥ 0.85: 必進人工審核（P0/P1）
- 0.7-0.85: 半自動審核（額外來源補證 + 二次模型檢查）

**誤判處理**:
- 審核員「誤判標記」必備
- 週期性模型回訓（每週）
- 動態白名單（僅限來源/主題類別級別）

**緊急下架**:
- 投訴進入 P0 隊列即刻暫停展示
- 24小時內必完成複審
- 維持下架則降低相似內容推薦權重

---

### 5. 證據鏈與存儲

**截圖存儲**:
- 雲端主存: S3/OSS（版本化、生命周期規則：90天熱存）
- 本地緩存: 7-14天熱素材

**日誌保留**:
- 標準: 90天
- 合規: 180-365天（商業合作或投訴頻發主題）

**監控指標**:
- 內容可信度: 雙來源覆蓋率、單來源發布比例、審核通過率
- 匹配質量: NLP-CV核心匹配通過率、人工介入率、審核平均時長
- 用戶反饋: 內容滿意度、投訴率、回訪率
- 成本與效能: 每篇內容生成成本、來源抓取延遲、API錯誤率、Fallback佔比
- 風險控制: 高風險觸發數、緊急下架次數、誤判率

## ⚠️ 邊界情況處理

### 全來源失效
- 降級為 Fallback + 降權曝光 + 明示標記
- 延長排程間隔 + 通知運維

### 用戶長期不互動
- 頻率降檔 + 以探索內容替代部分精準推送

### 模型誤判高峰
- 暫時放寬半自動審核門檻（0.75 → 0.8）
- 快速回訓後恢復
